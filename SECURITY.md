# Chromatic Browser Security Policy

**Chromatic Browser** © 2026 by **Scemercat1 or Joe** is committed to keeping users safe and maintaining a secure software ecosystem. This Security Policy explains how vulnerabilities can be reported, handled, and disclosed responsibly.

---

## Reporting a Vulnerability

If you discover a security issue in Chromatic Browser, please follow these steps:

1. Send a detailed report to **junethecat07@gmail.com** .  
2. Include the following information:
   - Steps to reproduce the issue
   - Version of Chromatic Browser affected
   - Platform/OS details
   - Any relevant screenshots, logs, or code snippets

> ✅ We encourage responsible disclosure. Do **not** exploit vulnerabilities for personal gain.

---

## Response and Timeline

- **Acknowledgment:** We will respond within **3 business days**.  
- **Assessment:** Security team reviews the report and verifies the issue.  
- **Resolution:** Fix will be prioritized based on severity. Critical vulnerabilities may be patched immediately.  
- **Communication:** Reporter may receive updates about the patch and credit in release notes if desired.

---

## Responsible Disclosure Guidelines

- Do **not** publicly disclose vulnerabilities before a patch is released.  
- Do **not** share exploit code with third parties.  
- Safe testing should be done in a controlled environment.
